# Simple Summary

This lecture demonstrates an **end‑to‑end production deployment** of an ML model using **AWS infrastructure**, fully automated and scalable.

### What this lecture achieves

You build a **real production architecture** where:

*   The model runs on **multiple EC2 instances**
*   Instances are created automatically using **Auto Scaling Group (ASG)**
*   Each instance is fully configured using a **user‑data script**
*   Traffic is routed through an **Application Load Balancer (ALB)**
*   Scaling happens automatically based on load
*   Costs are optimized (scale up when busy, scale down when idle)

***

### Key Components and Their Roles

| Component                          | Purpose                          |
| ---------------------------------- | -------------------------------- |
| **VPC**                            | Isolated network for resources   |
| **Subnets (2 AZs)**                | High availability                |
| **Internet Gateway + Route Table** | Public internet access           |
| **Security Group**                 | Open ports (22, 80)              |
| **Launch Template**                | Blueprint for EC2 instances      |
| **User Data Script**               | Auto‑configures model + API      |
| **Auto Scaling Group**             | Auto add/remove instances        |
| **Target Group**                   | Connects ASG to Load Balancer    |
| **Load Balancer**                  | Distributes traffic to instances |

***

### End‑to‑End Request Flow

1.  User sends request to **Load Balancer DNS**
2.  Load Balancer listener receives it on port **80**
3.  Request is forwarded to **Target Group**
4.  Target Group routes request to **ASG instances**
5.  EC2 instance serves request via:
    *   **Nginx → Gunicorn → Model API**
6.  ASG dynamically scales instances based on load

✅ Result: **scalable, cost‑effective, production‑grade model serving**

# Step‑by‑Step Lab Guide

> All commands are executed using **AWS CloudShell**  
> (No credential setup required)

***

## Part 1: Preparation

### Step 1: Open AWS CloudShell

*   Login to AWS Console
*   Select nearest region
*   Open **CloudShell**

Verify:

```bash
aws s3 ls
```

***

### Step 2: Fetch Ubuntu AMI ID

AMI differs per region.

```bash
aws ec2 describe-images \
--owners 099720109477 \
--filters "Name=name,Values=ubuntu/images/hvm-ssd/ubuntu-focal-20.04-amd64-server*" \
--query "Images[0].ImageId" \
--output text
```

Save it:

```bash
export AMI_ID=ami-xxxxxxxx
```

***

## Part 2: Networking Setup

### Step 3: Create VPC

```bash
aws ec2 create-vpc \
--cidr-block 10.0.0.0/16 \
--query "Vpc.VpcId" \
--output text
```

```bash
export VPC_ID=vpc-xxxxxxxx
```

***

### Step 4: Create Two Public Subnets

```bash
aws ec2 create-subnet --vpc-id $VPC_ID --cidr-block 10.0.1.0/24 --availability-zone us-east-1a
aws ec2 create-subnet --vpc-id $VPC_ID --cidr-block 10.0.2.0/24 --availability-zone us-east-1b
```

Save subnet IDs:

```bash
export SUBNET1_ID=subnet-xxxx
export SUBNET2_ID=subnet-yyyy
```

***

### Step 5: Internet Gateway & Routing

```bash
aws ec2 create-internet-gateway
export IGW_ID=igw-xxxx

aws ec2 attach-internet-gateway --vpc-id $VPC_ID --internet-gateway-id $IGW_ID
```

Create route table:

```bash
aws ec2 create-route-table --vpc-id $VPC_ID
export RTB_ID=rtb-xxxx
```

Add route:

```bash
aws ec2 create-route --route-table-id $RTB_ID --destination-cidr-block 0.0.0.0/0 --gateway-id $IGW_ID
```

Associate subnets:

```bash
aws ec2 associate-route-table --route-table-id $RTB_ID --subnet-id $SUBNET1_ID
aws ec2 associate-route-table --route-table-id $RTB_ID --subnet-id $SUBNET2_ID
```

✅ Subnets are now public

***

## Part 3: Security & Access

### Step 6: Create Security Group

```bash
aws ec2 create-security-group \
--group-name mlops-sg \
--description "Allow SSH and HTTP" \
--vpc-id $VPC_ID
```

```bash
export SG_ID=sg-xxxx
```

Allow ports:

```bash
aws ec2 authorize-security-group-ingress --group-id $SG_ID --protocol tcp --port 22 --cidr 0.0.0.0/0
aws ec2 authorize-security-group-ingress --group-id $SG_ID --protocol tcp --port 80 --cidr 0.0.0.0/0
```

***

## Part 4: Launch Template

### Step 7: Prepare User Data Script

```bash
git clone https://github.com/iam-veeramalla/Intent-classifier-model.git
cd intent-classifier-model
git checkout virtual-machines
```

Encode script:

```bash
export USER_DATA=$(base64 user-data.sh)
```

***

### Step 8: Create Key Pair

```bash
aws ec2 create-key-pair --key-name MLOpsKey --query "KeyMaterial" --output text > MLOpsKey.pem
chmod 400 MLOpsKey.pem
```

***

### Step 9: Create Launch Template

```bash
aws ec2 create-launch-template \
--launch-template-name MLOpsTemplate \
--launch-template-data "{
  \"ImageId\":\"$AMI_ID\",
  \"InstanceType\":\"t2.medium\",
  \"KeyName\":\"MLOpsKey\",
  \"SecurityGroupIds\":[\"$SG_ID\"],
  \"UserData\":\"$USER_DATA\"
}"
```

✅ Blueprint for auto‑created EC2s is ready

***

## Part 5: Load Balancing & Scaling

### Step 10: Create Target Group

```bash
aws elbv2 create-target-group \
--name mlops-target \
--protocol HTTP \
--port 80 \
--vpc-id $VPC_ID
```

```bash
export TG_ARN=arn:aws:elasticloadbalancing:...
```

***

### Step 11: Create Load Balancer

```bash
aws elbv2 create-load-balancer \
--name model-deployment-lb \
--subnets $SUBNET1_ID $SUBNET2_ID \
--security-groups $SG_ID \
--scheme internet-facing
```

```bash
export ALB_ARN=arn:aws:elasticloadbalancing:...
```

***

### Step 12: Create Listener

```bash
aws elbv2 create-listener \
--load-balancer-arn $ALB_ARN \
--protocol HTTP \
--port 80 \
--default-actions Type=forward,TargetGroupArn=$TG_ARN
```

***

### Step 13: Create Auto Scaling Group

```bash
aws autoscaling create-auto-scaling-group \
--auto-scaling-group-name mlops-asg \
--launch-template LaunchTemplateName=MLOpsTemplate \
--min-size 1 \
--max-size 3 \
--vpc-zone-identifier "$SUBNET1_ID,$SUBNET2_ID"
```

Attach target group:

```bash
aws autoscaling attach-load-balancer-target-groups \
--auto-scaling-group-name mlops-asg \
--target-group-arns $TG_ARN
```

✅ ASG + ALB integration complete
