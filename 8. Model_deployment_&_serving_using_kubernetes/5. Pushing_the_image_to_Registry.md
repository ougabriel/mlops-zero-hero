# ✅ Simple Summary

After building and testing the Docker image locally, the next step is to **push the model container image to a Model Registry**.  
A **Model Registry** is a centralized, versioned, auditable storage system for model artifacts—in this case, **container images**.

### Why a Model Registry is important:

*   ✅ Centralized storage
*   ✅ Versioning (v1, v2, etc.)
*   ✅ Auditing (who created/published an image)
*   ✅ RBAC (restrict who can push/delete)
*   ✅ Collaboration (QA, Devs, ML Engineers can all pull the image)
*   ✅ Needed for Kubernetes deployments (clusters must pull from a registry)

### Registries you can use:

*   **Docker Hub**
*   **Amazon ECR**
*   **Google Artifact Registry**
*   **GitHub Container Registry**
*   **Azure Container Registry**

In this lecture, Docker Hub is used.

You will:

1.  Create a Docker Hub account
2.  Generate a Personal Access Token
3.  Log in from your terminal
4.  Tag your local model image properly
5.  Push the image to Docker Hub
6.  Verify the image on Docker Hub

This makes the image publicly available (or private, depending on settings) so that **Kubernetes can pull it in the next lectures**.

***

# ✅ Step‑by‑Step Guide: Pushing the Model Image to a Registry (Docker Hub)

***

## ✅ Step 1: Create a Docker Hub Account

Go to:  
<https://hub.docker.com>

Create login + password  
(or organization account)

***

## ✅ Step 2: Create a Personal Access Token (PAT)

1.  Log in to Docker Hub
2.  Go to **Account Settings → Security**
3.  Click **New Access Token**
4.  Give it a name (e.g., `model-registry-token`)
5.  Set permissions (Admin recommended)
6.  Copy the token (you will not see it again)

***

## ✅ Step 3: Log in to Docker Hub from Terminal

```bash
docker login -u <dockerhub-username>
```

When prompted, paste your **Personal Access Token** as the password.

If successful:

    Login Succeeded

***

## ✅ Step 4: Tag Your Local Image Properly

Let’s assume your local image is:

    mlops-demo:latest

Tag it using:

```bash
docker tag mlops-demo:latest <dockerhub-username>/intent-classifier-model:v1
```

Example:

```bash
docker tag mlops-demo:latest abhishekf5/intent-classifier-model:v1
```

***

## ✅ Step 5: Push the Image to Docker Hub

```bash
docker push <dockerhub-username>/intent-classifier-model:v1
```

Example:

```bash
docker push abhishekf5/intent-classifier-model:v1
```

You’ll see the image layers upload one by one until completion.

***

## ✅ Step 6: Verify the Image on Docker Hub

1.  Go to Docker Hub
2.  Click on **Repositories**
3.  Open your repo (e.g., `intent-classifier-model`)
4.  You should see:
    *   The tag (`v1`)
    *   Pull instructions
    *   Last updated time

***

## ✅ Step 7: Pull the Image From Anywhere (Optional Test)

From any system (even AWS CloudShell):

```bash
docker pull <dockerhub-username>/intent-classifier-model:v1
```

This ensures the push was successful.

***

# ✅ End Result

You now have:

*   A **containerized ML model**
*   Pushed to **Docker Hub**
*   Versioned (v1)
*   Publicly or privately accessible
*   Ready for **Kubernetes deployment** in the next section
