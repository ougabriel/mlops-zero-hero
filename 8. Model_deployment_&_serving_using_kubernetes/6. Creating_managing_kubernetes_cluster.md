# ✅ **Simple Summary**

This lecture teaches you **how to create, manage, and understand a Kubernetes cluster** for model deployment — especially for production‑grade MLOps workflows.

It covers two major areas:

***

## ✅ **Part 1 — Creating a Kubernetes Cluster**

You learn that there are **two ways** to run Kubernetes clusters:

### **1. Local Kubernetes Cluster (Free)**

Tools:

*   **kind** (Kubernetes-in-Docker)
*   **minikube**
*   **microk8s**

Pros:

*   Free
*   Good for demos & learning
*   Quick to set up

Cons:

*   Ingress does not expose apps to the real internet
*   Only accessible inside your machine/network
*   Not production-ready

**Kind is recommended** for local testing because:

*   Creates a cluster in <2 minutes
*   Runs the cluster inside Docker
*   Lightweight and resource-efficient

***

### **2. Cloud-Hosted Kubernetes Cluster (Production)**

Tools:

*   **EKS** (AWS Elastic Kubernetes Service)
*   **AKS** (Azure Kubernetes Service)
*   **GKE** (Google Kubernetes Engine)
*   OpenShift (Enterprise Kubernetes)
*   Self‑hosted Kubernetes (DIY on VMs)

Pros:

*   Production-ready
*   Auto‑scaling, load balancing, networking
*   Real ingress with public endpoints

Cons:

*   Costs money (but EKS demo costs \~USD 1–5 per day)

***

### ✅ For this course, the instructor uses **AWS EKS**

Because:

*   Easy to set up
*   Students worldwide can create AWS accounts easily
*   Costs are low if deleted immediately after the demo

You prepare an EKS cluster using:

*   **AWS CloudShell**
*   **AWS CLI**
*   **kubectl**
*   **eksctl**

CloudShell advantage:

*   AWS CLI & kubectl already installed
*   No local machine setup required
*   Automatically authenticated with your AWS credentials

The cluster is created using:

```bash
eksctl create cluster \
  --name demo-cluster \
  --region ap-south-1 \
  --version 1.32 \
  --nodegroup-name ng \
  --node-type t3.medium \
  --nodes-min 1 \
  --nodes-max 3
```

The cluster creation takes **15–20 minutes**.

You can watch EKS cluster creation progress in:
**AWS Console → CloudFormation → Events**

## ✅ **Part 2 — How Organizations Share a Kubernetes Cluster**

A very important conceptual part (also asked in interviews!).

In most companies:

> **One Kubernetes cluster is shared across many teams using Namespaces.**

### **Organization structure example**

*   Payments Team
*   Shipping Team
*   Login/Authentication Team
*   Product Catalog Team
*   Fraud Detection Team
*   MLOps Team

Each team gets its own namespace:

    payments-namespace
    shipping-namespace
    login-namespace
    catalog-namespace
    mlops-namespace

### ✅ **Key Governance Mechanisms**

1.  **Namespaces** → Logical isolation
2.  **ResourceQuota** → Limit CPU/RAM per namespace
3.  **RBAC** → Limit access so each team only manages its namespace
4.  **Requests & Limits** → Resource boundaries per pod

This:

*   Prevents one team from taking all CPU/RAM
*   Reduces cost
*   Ensures stability
*   Allows many teams to deploy safely on one cluster

### ✅ **Interview Answer**

> “We maintain one Kubernetes cluster per environment (dev/staging/prod).  
> Each team gets a dedicated namespace with a ResourceQuota.  
> RBAC restricts access to each namespace, and teams configure resource requests and limits for their pods.”

# ✅ **Step‑by‑Step Lab Guide: Creating an EKS Cluster Using CloudShell**

Follow these exact steps.

***

## ✅ Step 1 — Open AWS CloudShell

AWS Console → top-right → **CloudShell**

Verify AWS CLI works:

```bash
aws s3 ls
```

***

## ✅ Step 2 — Check Prerequisites

### Check kubectl

```bash
kubectl version --client
```

### Check eksctl

```bash
eksctl version
```

If eksctl is missing, install using:

```bash
curl --silent --location "https://github.com/eksctl-io/eksctl/releases/latest/download/eksctl_$(uname -s)_amd64.tar.gz" | tar xz -C /tmp
sudo mv /tmp/eksctl /usr/local/bin
```

***

## ✅ Step 3 — Create the Kubernetes Cluster

Example command (replace region if needed):

```bash
eksctl create cluster \
  --name demo-cluster \
  --region ap-south-1 \
  --version 1.32 \
  --nodegroup-name demo-nodes \
  --node-type t3.medium \
  --nodes 1 \
  --nodes-min 1 \
  --nodes-max 3
```

⏳ **Wait 15–20 minutes**

***

## ✅ Step 4 — Verify Cluster Creation

Update kubeconfig:

```bash
aws eks update-kubeconfig --name demo-cluster --region ap-south-1
```

Check active cluster:

```bash
kubectl config current-context
```

Expected:

    arn:aws:eks:ap-south-1:XXXX:cluster/demo-cluster

Check nodes:

```bash
kubectl get nodes
```

✅ If you see nodes → cluster is ready.

***

## ✅ Step 5 — How to Delete the Cluster (Important to Avoid Costs)

Once done:

```bash
eksctl delete cluster --name demo-cluster --region ap-south-1
```
