***

## Core Concept: Docker Lifecycle (Very Important)

This lecture reinforces the **three stages of Docker**:

1.  **Dockerfile**
    *   Blueprint that defines how your application + model should run
    *   Created by DevOps / MLOps engineers

2.  **Docker Image**
    *   Built from the Dockerfile
    *   A portable snapshot containing:
        *   OS base image
        *   System dependencies
        *   Python dependencies
        *   Model + API code

3.  **Docker Container**
    *   A **running instance** of the Docker image
    *   This is where:
        *   The Flask/Gunicorn API runs
        *   Model inference happens
        *   Users send requests to `/predict`

✅ Once the container is running, **model serving is active**

***

# Step‑by‑Step Lab Guide: Running & Testing Docker Image Locally

***

## Prerequisites

*   Docker Desktop installed and running
    *   macOS → Docker Desktop (DMG)
    *   Windows → Docker Desktop (EXE)
*   Git repository cloned
*   Switched to the **Kubernetes branch**

```bash
git checkout kubernetes
git pull
```

***

## Step 1: Verify Docker Is Running

Check Docker engine:

```bash
docker --version
docker info
```

✅ Docker must be running before continuing

***

## Step 2: Build the Docker Image

From the directory containing the `Dockerfile`:

```bash
docker build -t mlops-demo:latest .
```

### Explanation

*   `docker build` → builds an image
*   `-t` → tag for image
*   `mlops-demo` → image name
*   `latest` → version tag
*   `.` → Dockerfile is in current directory

***

## Step 3: Observe the Build Process

Docker will execute:

1.  Pull Python base image
2.  Set environment variables
3.  Copy `requirements.txt`
4.  Install dependencies
5.  Copy source code
6.  Run `model/train.py`
7.  Configure the CMD (Gunicorn)

✅ If no errors appear → image build succeeded

***

## Step 4: Verify Image Exists Locally

```bash
docker images | grep mlops-demo
```

✅ You should see:

    mlops-demo   latest   <image-id>

***

## Step 5: Run the Docker Container

```bash
docker run -d -p 6000:6000 mlops-demo:latest
```

### Explanation

*   `-d` → run in background (detached)
*   `-p 6000:6000` → map container port to local port
*   Container port `6000` → Gunicorn API
*   Local port `6000` → where you send requests

✅ Container is now running

***

## Step 6: Verify Container Is Running

```bash
docker ps
```

✅ You should see:

*   Container ID
*   Image: `mlops-demo:latest`
*   Port mapping: `0.0.0.0:6000->6000`

***

## Step 7: Test the Model API Using curl

```bash
curl -X POST http://localhost:6000/predict \
-H "Content-Type: application/json" \
-d '{"text":"Hi"}'
```

✅ Expected output:

```json
{"intent":"greeting"}
```

This confirms:

*   Container is running
*   Gunicorn is serving the API
*   Model inference works inside Docker

***

## Step 8: (Optional) View Container Logs

```bash
docker logs <container-id>
```

Useful for debugging startup or inference issues.

***

## Step 9: Stop and Remove the Container

```bash
docker ps | grep mlops-demo
```

Copy the container ID, then:

```bash
docker rm -f <container-id>
```

***

## Step 10: (Optional) Remove the Docker Image

⚠️ Only do this if you don’t need the image anymore:

```bash
docker rmi mlops-demo:latest
```
