# Summary: Deploying and Serving Models in SageMaker AI

This lecture explains how to deploy a machine learning model in SageMaker AI and how to serve it for inference. It builds on the previous lecture, where the model was created and stored in an S3 bucket. Deployment in SageMaker requires additional preparation because SageMaker cannot directly deploy a raw pickle (`.pkl`) file.

***

## 1. Why SageMaker Cannot Deploy `.pkl` Files Directly

*   Many data scientists save models using scikit-learn, which produces `.pkl` files.
*   SageMaker does not deploy raw `.pkl` models.
*   SageMaker expects the model artifact to be a `.tar.gz` archive containing:
    *   The model file (`.pkl`)
    *   An inference script (`inference.py`)

Therefore, two prerequisites must be completed before deployment:

### Prerequisite 1: Create `inference.py`

A Python script that defines four required functions:

*   `model_fn` – loads the model from the artifact
*   `input_fn` – formats incoming payload
*   `predict_fn` – performs the prediction
*   `output_fn` – returns the prediction response

This script follows the exact structure expected by SageMaker containers.

### Prerequisite 2: Create a `.tar.gz` archive

Bundle:

*   `iris_model.pkl`
*   `inference.py`

Then upload this `.tar.gz` file to the S3 bucket (model registry).

***

## 2. Creating the `.tar.gz` Model Artifact in SageMaker Studio

### Steps:

1.  Create `inference.py` inside SageMaker Studio.
2.  Use a small Python script to package:
    *   `iris_model.pkl`
    *   `inference.py`
        into `model.tar.gz` using the `tarfile` module.
3.  Upload the archive to S3 using boto3 (`s3.upload_file()`).

After uploading, S3 now contains the deployable model artifact.

***

## 3. Deploying the Model in SageMaker

There are two main deployment methods:

***

## Method 1: Deployment Through Python Code

Inside Jupyter Notebook:

1.  Import `sagemaker.sklearn.model.SKLearnModel`.
2.  Provide:
    *   `model_data` (S3 path of `.tar.gz`)
    *   role ARN (the SageMaker execution role)
    *   entry point (`inference.py`)
    *   framework version (scikit-learn version)
3.  Call `.deploy()` with:
    *   instance type (for example, `ml.t2.medium`)
    *   instance count (usually 1 for demos)

SageMaker will:

*   Create an endpoint configuration
*   Launch an EC2 instance
*   Pull a prebuilt container for scikit-learn
*   Load your model and inference script inside the container
*   Host the endpoint for inference

When deployment completes, an inference endpoint appears in SageMaker.

***

## Method 2: Deployment Through the SageMaker Console (UI)

If the user does not prefer Python scripts:

1.  Go to SageMaker > Models > Deployable Models
2.  Create a new deployable model:
    *   Choose framework: scikit-learn
    *   Provide framework version
    *   Provide S3 model artifact path
    *   Provide IAM role ARN
3.  After model creation, click “Deploy”
4.  Choose instance type and instance count

This achieves the same result as the Python method.

***

## 4. Understanding the Deployment Architecture

Once deployed:

1.  SageMaker launches an EC2 instance.
2.  A prebuilt container for the selected ML framework is pulled.
3.  Your model and `inference.py` are placed inside the container.
4.  SageMaker creates an inference endpoint.
5.  End users send requests to the endpoint, which:
    *   Passes inputs to your `input_fn`
    *   Runs predictions using `predict_fn`
    *   Returns outputs via `output_fn`

***

## 5. Performing Inference

Using the SageMaker console:

1.  Go to the deployed endpoint.
2.  Open “Test Inference”.
3.  Provide JSON input payload matching your model features.
4.  Send the request.
5.  View the predicted output returned by the model.

This verifies that the model is properly deployed and served.

***

## 6. Real-World Considerations

*   SageMaker endpoints can be fronted with API Gateway, Load Balancers, or Lambda for production use.
*   The deployment process may take several minutes, especially for larger models.
*   Practicing with multiple models helps build clarity and confidence.

***

# Final Summary

To deploy and serve a model in SageMaker:

1.  Create `inference.py` (required for all custom deployments).
2.  Package `model.pkl` and `inference.py` into a `.tar.gz` file.
3.  Upload the archive to S3.
4.  Deploy using either:
    *   Python SDK (`SKLearnModel.deploy()`), or
    *   SageMaker UI
5.  Test inference through the endpoint.

This completes the end‑to‑end process of model creation, packaging, deployment, and serving using SageMaker AI.
