# Summary: How to Safely Delete SageMaker Resources

This lecture explains the correct procedure for deleting SageMaker AI resources to avoid unnecessary charges. SageMaker domains, user profiles, and associated applications (spaces, Jupyter instances) must be removed in the correct order. You cannot delete a domain directly unless all dependent resources are deleted first.

***

## 1. You Cannot Delete a Domain Directly

SageMaker prevents domain deletion if:

*   User profiles exist
*   Spaces exist
*   Applications or running instances exist

Therefore, deletion must be done in a specific sequence.

***

## 2. Step-by-Step Deletion Process

### Step 1: Delete Resources (Applications)

1.  Open AWS Console → SageMaker AI
2.  Go to the specific domain
3.  Open **Resources**
4.  Stop and delete running applications (for example, JupyterLab instances)
5.  This ensures no compute is still running, preventing charges

***

### Step 2: Delete Spaces

1.  In the domain view, go to **Space Management**
2.  Delete all spaces
3.  Spaces must be removed before user profiles can be deleted

***

### Step 3: Delete User Profiles

1.  Go to **User Profiles** inside the domain
2.  Delete each user profile
3.  If multiple profiles exist, delete all of them

***

### Step 4: Delete the Domain

Once all the above dependencies are removed:

1.  Return to the main domain page
2.  Scroll down to the bottom
3.  The "Delete Domain" button will now be enabled
4.  Confirm deletion

If the delete button is greyed out, it means:

*   A resource, space, or profile still exists

***

## 3. Why This Is Important

If you leave SageMaker resources active:

*   Domain hosting charges continue
*   Notebook instances may continue billing
*   Studio resources can accumulate significant costs

Deleting domains and all dependencies prevents ongoing charges.

***

# Final Note

Always delete SageMaker domains when finished practicing or experimenting. Leaving the domain running can cost far more than the price of the course.
