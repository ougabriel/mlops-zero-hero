# Step‑by‑Step Guide for an MLOps Engineer Using AWS SageMaker

This guide assumes an MLOps engineer is enabling a data science team to build, train, version, deploy, and serve models in SageMaker.

***

# 1. Understand the SageMaker Structure

SageMaker AI operates with these key components:

1.  **Domain** – the workspace that hosts Studio, notebooks, pipelines, MLflow, etc.
2.  **User Profiles** – individual workspaces inside a domain.
3.  **Execution Roles** – IAM roles used by Studio applications and services.
4.  **Model Registry** – typically an S3 bucket storing model artifacts.
5.  **Endpoints** – deployed models running behind scalable infrastructure for inference.

***

# 2. Prepare the Infrastructure (VPC, Subnets, Roles)

### 2.1 Get VPC and Subnets

You need to know which VPC and subnets SageMaker will run in.

From CLI:

    aws ec2 describe-vpcs --query "Vpcs[?IsDefault==true].VpcId"
    aws ec2 describe-subnets --filters Name=vpc-id,Values=<vpc-id>

Use the **default VPC** unless your organization requires a custom network.

***

# 3. Create Execution Roles for SageMaker

SageMaker Studio and deployed models need a role with permissions to interact with AWS services.

### 3.1 Create trust policy (trust.json)

    {
      "Version": "2012-10-17",
      "Statement": [
        {
          "Effect": "Allow",
          "Principal": {"Service": "sagemaker.amazonaws.com"},
          "Action": "sts:AssumeRole"
        }
      ]
    }

### 3.2 Create the role

    aws iam create-role \
      --role-name SageMakerExecutionRole \
      --assume-role-policy-document file://trust.json

### 3.3 Attach required permissions

    aws iam attach-role-policy \
      --role-name SageMakerExecutionRole \
      --policy-arn arn:aws:iam::aws:policy/AmazonSageMakerFullAccess

    aws iam attach-role-policy \
      --role-name SageMakerExecutionRole \
      --policy-arn arn:aws:iam::aws:policy/AmazonS3FullAccess

This role will be assigned to the domain.

***

# 4. Create the SageMaker Domain

Using CLI:

    aws sagemaker create-domain \
      --domain-name MLOpsDemoDomain \
      --auth-mode IAM \
      --default-user-settings ExecutionRole=<role-arn> \
      --vpc-id <vpc-id> \
      --subnet-ids <subnet-1> <subnet-2> \
      --app-network-access-type VpcOnly \
      --region <region>

Wait until the domain status becomes **InService**.

***

# 5. Create User Profiles

These profiles represent each data scientist or ML engineer.

    aws sagemaker create-user-profile \
      --domain-id <domain-id> \
      --user-profile-name AliceProfile

For real‑world setups, configure **ABAC** so IAM users can only open their own profiles.  
For model‑building demonstrations, you can skip ABAC and use the admin account.

***

# 6. Provide Access to SageMaker Studio

The user logs in:

*   AWS Console → SageMaker AI → Domain → User Profile
*   Launch Studio

Inside Studio, they gain access to JupyterLab, notebooks, etc.

***

# 7. Create the Model (Data Scientist Workflow)

Inside JupyterLab:

### 7.1 Create a notebook and import requirements

Example:

    from sklearn.datasets import load_iris
    from sklearn.tree import DecisionTreeClassifier
    import joblib

### 7.2 Train the model

    X, y = load_iris(return_X_y=True)
    clf = DecisionTreeClassifier()
    clf.fit(X, y)
    joblib.dump(clf, "iris_model.pkl")

### 7.3 Upload model to S3 (model registry)

    import boto3
    s3 = boto3.client("s3")
    s3.upload_file("iris_model.pkl", "<bucket>", "model-artifacts/iris_model.pkl")

At this point, the model is registered.

***

# 8. Prepare the Model for Deployment in SageMaker

SageMaker requires a **model.tar.gz** containing:

*   inference.py
*   iris\_model.pkl

### 8.1 Create inference.py

Must contain these functions:

*   model\_fn
*   input\_fn
*   predict\_fn
*   output\_fn

This script defines how the model loads and handles requests.

### 8.2 Package the model

Inside Jupyter:

    import tarfile

    with tarfile.open("model.tar.gz", "w:gz") as tar:
        tar.add("iris_model.pkl")
        tar.add("inference.py")

### 8.3 Upload model archive to S3

    s3.upload_file("model.tar.gz", "<bucket>", "model-artifacts/model.tar.gz")

***

# 9. Deploy the Model

There are two deployment approaches.

***

## Method 1: Deployment Using Python in Notebook

    from sagemaker.sklearn.model import SKLearnModel

    model = SKLearnModel(
        model_data="s3://<bucket>/model-artifacts/model.tar.gz",
        entry_point="inference.py",
        role="<execution-role-arn>",
        framework_version="1.2"
    )

    endpoint = model.deploy(
        instance_type="ml.t2.medium",
        initial_instance_count=1
    )

This automatically:

*   Creates endpoint configuration
*   Launches an EC2 instance
*   Loads the model
*   Starts a container
*   Exposes an HTTPS endpoint

***

## Method 2: Deployment Using the SageMaker Console

1.  Go to SageMaker → Models → Deployable Models
2.  Create new model
3.  Select "scikit-learn" as the framework
4.  Provide:
    *   S3 model artifact
    *   Execution role
5.  Click “Deploy”
6.  Choose instance type
7.  Launch the endpoint

***

# 10. Test the Endpoint (Inference)

From the console:

*   Go to Endpoints
*   Select your endpoint
*   Choose "Test Inference"
*   Provide JSON input matching model features
*   Send request
*   View prediction response

***

# 11. Secure the Endpoint (Real‑World Practice)

For production use:

*   Use API Gateway
*   Use a Load Balancer for high traffic
*   Add rate limits
*   Add VPC-only access if needed
*   Use IAM authorization or custom auth layers

***

# 12. Clean Up Resources (Prevent Unnecessary Costs)

When finished:

### Delete notebook instances (Apps)

*   SageMaker → Domain → Resources
*   Stop and delete apps (JupyterLab sessions)

### Delete spaces

*   Domain → Spaces → Delete

### Delete user profiles

*   Domain → User Profiles → Delete

### Delete the domain

*   Domain → Delete

Failing to clean up will result in ongoing charges.

***

# 13. Summary of the Complete Workflow for MLOps Engineers

1.  Prepare environment (VPC, subnets, IAM roles).
2.  Create SageMaker domain.
3.  Create user profiles.
4.  Set up ABAC for access control (real‑world setups).
5.  Provide SageMaker Studio access to data scientists.
6.  Data scientists train models in Jupyter.
7.  Store trained models in S3 (model registry).
8.  Create inference script (`inference.py`).
9.  Package model (model.tar.gz).
10. Upload packaged model to S3.
11. Deploy model via Python or SageMaker UI.
12. Serve predictions using endpoint.
13. Add API Gateway or LB in production if needed.
14. Delete all SageMaker resources when done.
