# Summary: Practical Setup of SageMaker AI Using AWS CLI

This lecture demonstrates how to set up Amazon SageMaker AI in a real environment using AWS CLI. It implements the three-step architecture previously explained: domain creation, user profile creation, and ABAC enforcement to restrict access.

***

## 1. Prerequisites

*   AWS CLI must be installed and configured.
*   Alternatively, AWS CloudShell can be used.
*   All SageMaker resources must run inside a VPC, so VPC ID and subnet IDs are required before creating a domain.

***

## 2. Preparing VPC Information

*   Retrieve the default VPC ID using `aws ec2 describe-vpcs`.
*   Retrieve subnet IDs inside the chosen VPC.
*   These values will be used when creating the domain.

***

## 3. Step 1: Creating the SageMaker Domain

### 3.1 Create an IAM Role (Execution Role)

*   SageMaker Studio applications need an execution role to interact with AWS resources such as EC2, EBS, S3, ECR, and CloudWatch.
*   Create a trust policy file (trust.json) that allows the `sagemaker.amazonaws.com` principal to assume the role.
*   Create the IAM role using `aws iam create-role`.
*   Attach the permission policy `AmazonSageMakerFullAccess` to the role.
*   Retrieve the role ARN for use in domain creation.

### 3.2 Create the Domain

*   Run `aws sagemaker create-domain` with:
    *   Domain name
    *   Authentication mode = IAM
    *   VPC ID
    *   List of subnet IDs
    *   Execution role ARN
    *   Network access mode = VPC only
*   Domain creation outputs Domain ID and Domain ARN, but the domain enters a pending state for several minutes.
*   Wait in the AWS Console until the domain status becomes “InService”.

***

## 4. Step 2: Creating User Profiles

### 4.1 Create User Profile with Tags (ABAC Preparation)

*   Use `aws sagemaker create-user-profile` with:
    *   Domain ID
    *   User Profile Name (example: AliceProfile)
    *   Tags: A key-value pair (example: `studio-user-id = alice123`)
*   These tags will be used later to enforce ABAC.

### 4.2 Create Additional Profiles

*   Create a second profile (example: AbhishekProfile) for testing access restrictions.
*   Confirm both profiles appear under the domain in the AWS Console.

***

## 5. Step 3: Implementing ABAC (Attribute-Based Access Control)

### 5.1 Create IAM User for Alice

*   If Alice does not already have an IAM user, create one with `aws iam create-user`.
*   Add the same tag used for the user profile:
    `studio-user-id = alice123`
*   This matches the profile tag, which ABAC requires.

### 5.2 Attach ABAC-Controlled Policy to Alice

*   Create an IAM policy JSON file (SageMakerABAC.json) containing:
    *   Statement 1: Allows general SageMaker reading actions  
        (list domains, list profiles, describe profiles, etc.)
    *   Statement 2: ABAC condition  
        Allows access only to SageMaker resources whose tags match the IAM user’s tag.  
        Example:  
        `"Condition": { "StringEquals": { "aws:ResourceTag/studio-user-id": "${aws:PrincipalTag/studio-user-id}" } }`

*   Create the policy using `aws iam create-policy`.

*   Attach the policy to Alice’s IAM user using `aws iam attach-user-policy`.

### 5.3 Enable Console Login for Alice

*   Enable AWS console sign-in for the IAM user and generate a password.
*   Sign in using a private browsing window.

***

## 6. Verification of ABAC

### Expected behavior:

1.  Alice can access the SageMaker service.
2.  Alice can see the domain and the list of user profiles.
3.  Alice cannot open or launch Studio for any profile except her own.
    *   Attempting to open Abhishek’s profile results in an AccessDenied error.
4.  Alice can successfully launch Studio for the AliceProfile.

This confirms ABAC is correctly implemented.

***

## 7. Optional: Generating a Presigned Studio URL

*   SageMaker supports generating user-specific presigned Studio URLs using:
    `aws sagemaker create-presigned-domain-url`
*   This link gives direct access to the user’s Studio environment.
*   Recommended only as a secondary method because links can be lost; console access is more reliable.

***

# Final Summary

The lecture demonstrates full real-world setup of SageMaker AI using automation-focused methods. The process follows three critical phases:

1.  **Create the domain** using AWS CLI and attach an execution role with SageMaker permissions.
2.  **Create user profiles**, ensuring each profile is tagged with a unique identifier.
3.  **Implement ABAC** by tagging corresponding IAM users and enforcing tag-based IAM policies, ensuring each user can only access their own Studio profile.

This is the production-grade approach used in organizations and the expected explanation in interviews when discussing SageMaker AI setup.
